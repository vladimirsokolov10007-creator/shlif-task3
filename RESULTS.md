# Результаты — Task-3 «Скажи мне, кто твой шлиф»

Классификация шлифов по сортам руды + слабая сегментация панорам скользящим окном.
Обучение: `efficientnet_b0`, вход 320px, focal-loss с весами классов, 5-fold
StratifiedCV, CPU. Прогон: `runs/20260703_230941/` (см. `summary.json`).

### Соответствие целям ТЗ

- **«F1 ≥ 90% по срастаниям».** `regular` (обычные срастания) = **0.90 — цель достигнута**;
  `fine` (тонкие срастания) = 0.76 — **не дотягивает**. Класс `fine` путается с `regular`/`talc`.
- **«Отклонение ≤ 3% по доле талька».** Доля `talc` по панорамам считается (ниже),
  но **эталонного состава панорам в данных нет — сверять не с чем**. Метрика остаётся
  нереализуемой до появления референса. См. раздел «Ограничения».
- `refractory` (F1 0.42) — слабый из-за малой выборки (68 фото) и того, что классa нет
  в исходном ТЗ. Тянет macro-F1 вниз.

## 3. Ограничения (честно)

1. **Доминирование `talc` (0.58–0.96) на панорамах — вероятное смещение модели, не факт.**
   Панорамы содержат много матрикса/фона, визуально ближе к «оталькованным» фото; плюс
   классификатор обучен на фото-кропах, а не на тайлах панорам (domain shift). Без разметки
   панорам проверить нельзя — цифры трактовать как ориентир, не как истину.
2. **`6.jpg` и `7.jpg` — точный дубликат** (байт-в-байт идентичны, одинаковый MD5
   `5e26b6a0…`), поэтому и доли совпадают. Фактически уникальных панорам 13, а не 14.
3. **Нет пиксельных масок** → это классификация + слабая сегментация, а не обучаемая U-Net
   (см. `README.md`). Пиксельные F1/mIoU из исходного ТЗ неприменимы к этим данным.
4. **CPU-обучение** ограничило модель (`efficientnet_b0`, 320px). На GPU реально поднять
   и качество, и разрешение тайлинга.

## 4. Как воспроизвести

```bash
python -m src.data.extract   --config configs/config.yaml
python -m src.data.index     --config configs/config.yaml
python -m src.train          --config configs/config.yaml
python -m src.inference --checkpoint checkpoints/efficientnet_b0_fold3.pt --all-panoramas
```

## 5. Что улучшать дальше

- Дать эталонные доли по панорамам → включить метрику «отклонение по доле талька».
- Разметить 2–3 панорамы пиксельно → перейти к настоящей U-Net сегментации.
- TTA + ансамбль 5 фолдов на инференсе (веса уже есть в `checkpoints/`).
- Побороть `fine`/`refractory`: oversampling, сильнее фокус-веса, доп. аугментации.
- GPU: `efficientnet_b3`, вход 512px, полноразмерный тайлинг.
