# Шлиф · Task-3 «Скажи мне, кто твой шлиф» (Норникель)

Классификация панорамных OM-снимков шлифов по **сортам руды** и слабая
сегментация панорам скользящим окном.

> **Важно про данные.** В предоставленном архиве **нет пиксельных масок** —
> разметка только на уровне изображения (папка = сорт руды), а панорамы не
> размечены. Поэтому задача решается как **классификация фото** + **тайловый
> инференс панорам** (карта классов и доли по площади), а не как обучаемая
> пиксельная сегментация U-Net. Подробности — в разделе «Данные».

## Классы

| Папка в данных | Класс (id) | Смысл |
|---|---|---|
| оталькованные / Оталькованные руды / Области оталькования | `talc` (0) | тальк |
| рядовые / Рядовые руды | `regular` (1) | обычные срастания |
| тонкие | `fine` (2) | тонкие срастания |
| Труднообогатимые руды | `refractory` (3) | труднообогатимые |

Сопоставление папок и классов задаётся в `configs/config.yaml → data.class_map`
(можно объединять/переименовывать классы без правки кода).

## Данные

- **Панорамы**: 14 больших JPG, 126–574 Мп (13330–27025 × 9347–21227 px). Не
  размечены — целевой объект инференса.
- **Фото руд по сортам** (ч1 + ч2): ~1221 снимок (в осн. 2272×1704),
  разложены по папкам-сортам. Дисбаланс: `regular` ≫ `refractory`.
- Формат — JPG (один BMP). Гигапиксельных TIFF нет, поэтому `pyvips/tifffile`
  не используются. Имена внутри архива — в кодировке CP866 (чинятся при распаковке).

## Установка

```bash
# CPU (как в текущем окружении):
pip install torch torchvision --index-url https://download.pytorch.org/whl/cpu
pip install -r requirements.txt
# GPU: замените индекс на .../whl/cu121
```

Проект тестировался на Python 3.14 (CPU-сборка torch 2.12).

## Пайплайн

```bash
# 1. Распаковать архив (чинит CP866-имена) -> data/raw
python -m src.data.extract --config configs/config.yaml

# 2. Индексация + EDA-отчёт -> data/processed/index.csv
python -m src.data.index --config configs/config.yaml

# 3. Обучение (5-fold StratifiedCV, per-class F1 + macro-F1)
python -m src.train --config configs/config.yaml            # все фолды
python -m src.train --fold 0 --epochs 5                     # быстрый прогон

# 4. Инференс по панорамам (карта классов + доли + оверлей)
python -m src.inference --checkpoint checkpoints/efficientnet_b0_fold3.pt --all-panoramas

# 5. (опц.) Демо
pip install streamlit && streamlit run app/streamlit_app.py
```

## Веса и данные

Репозиторий содержит **только код и документацию**. Данные (~2.8 ГБ) и веса
моделей в git не хранятся (см. `.gitignore`) — их нужно получить отдельно:

- **Данные.** Исходный архив `Задача 3. Скажи мне, кто твой шлиф.zip` положите в
  корень проекта и распакуйте шагом 1 пайплайна (`src.data.extract`). Имена
  внутри архива в CP866 чинятся автоматически.
- **Веса.** Обученный чекпоинт (лучший фолд, `efficientnet_b0_fold3.pt`, ~45 МБ)
  публикуется через **GitHub Release**:
  `https://github.com/<owner>/<repo>/releases` → скачать в `checkpoints/`.
  Либо обучите с нуля шагом 3 (5-fold, ~4.5 ч на CPU).

> Результаты 5-fold CV и состав панорам — в [RESULTS.md](RESULTS.md).

## Структура

```
configs/config.yaml     конфиг (пути, классы, гиперпараметры, тайлинг) — без хардкодов
src/
  data/extract.py       распаковка zip + починка CP866-имён
  data/index.py         обход data/raw -> index.csv + EDA-отчёт
  data/dataset.py       torch Dataset + аугментации (torchvision.transforms.v2)
  data/tiling.py        тайлинг панорам + блендинг вероятностей по overlap
  model.py              классификатор (torchvision backbone) + focal/CE loss
  train.py              5-fold CV, метрики, чекпоинты, early stopping, логи
  inference.py          инференс панорамы -> карта классов + доли + оверлей
  utils.py              конфиг, сид, логгер, метрики, CP866-хелпер
notebooks/01_eda.ipynb  EDA
app/streamlit_app.py    демо-UI
```

## Метрики и целевые значения

- **per-class F1** и **macro-F1** по фолдам (в логе и `runs/*/summary.json`).
- Цель ТЗ *F1 ≥ 90 % по срастаниям* оценивается на отложенных фото (классы
  `regular`, `fine`).
- Цель *отклонение ≤ 3 % по доле талька*: инференс выдаёт долю `talc` по
  площади панорамы, но **сверять не с чем** — эталонного состава панорам в
  данных нет. Если появятся эталонные доли, сравнение добавляется тривиально
  (см. `outputs/panorama_fractions.csv`).

## Замечания по окружению

- **CPU-only.** Дефолт — лёгкий `efficientnet_b0`, вход 320 px. Тяжёлые
  бэкбоны/полноразмерный тайлинг практичны только на GPU (поменяйте
  `train.backbone` и переустановите torch под CUDA).
- Панорамы читаются по одной; для скорости есть `inference.read_downsample`.
